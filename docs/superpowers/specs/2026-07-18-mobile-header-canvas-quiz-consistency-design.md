# 手機頁籤、Canvas 避撞與例題連線一致性設計

## 目標

修正手機瀏覽器中的三類版面問題，並讓定比定律與倍比定律首頁採用一致的首次載入風格：

1. 四個頂部頁籤不得超出應用程式容器。
2. 定比定律與倍比定律首頁皆直接呈現正式第一步。
3. 所有窄版 Canvas 步驟不得出現段落、結論、表格或圖形互相重疊。
4. 所有定比例題的 `D`、`SiC` 與「矽 Z 克」節點使用一致且正確的連接錨點。

## 根因

- 頁籤按鈕強制不換行，頁籤列與版權按鈕又共處同一個 Flex 列；外層即使允許換行，頁籤列本身仍以內容寬度撐出容器。
- 倍比定律 HTML 的首次靜態標題與說明仍是舊的「開始探索」內容，與 JavaScript 載入後的正式第一步及定比定律首頁不一致。
- 部分 Canvas 以固定 Y 座標分別放置換行段落、表格和底部結論；窄畫布中文字增加行數後，這些區域會交疊。
- 例題關係圖使用個別硬編碼連線端點，沒有從「矽 Z 克」矩形的實際邊界計算上下中央錨點。

## 設計

### 手機頁籤

- 桌機與平板維持目前單列頁籤外觀。
- 手機斷點內將 `.soil-tabs` 改為兩欄 Grid，形成 2×2 排列。
- 頁籤列佔滿容器可用寬度，按鈕可縮小且文字置中；四個入口必須同時可見。
- 手機版權按鈕獨立排列在頁籤下方，不參與頁籤寬度計算。
- 四個正式頁面共用相同 CSS，不在各頁複製不同規則。

### 首頁一致性

- 以定比定律目前的正式第一步為互動基準。
- 倍比定律 HTML 的首次標題、說明與 JavaScript `stepTexts[0]` 完全一致，避免載入前後跳動。
- 倍比定律雙面板使用與定比定律相同的可縮 Grid 軌道與手機堆疊斷點。
- 保留兩套課程各自的教學內容與步驟數，不改變資料或互動邏輯。

### Canvas 文字避撞

- 盤點定比定律及倍比定律所有公開步驟與手機 compact renderer。
- 每個窄版 Canvas 分成主要內容區與保留的底部結論區。
- 換行文字先計算最後一行 baseline，再依最小間距安排下一個區塊。
- 表格與主要圖形的可用高度截止於底部結論區上緣，不得用結論文字覆蓋表格底線。
- 同一類步驟共用相同的底部錨定方式，不只修正截圖中的第 5、6 步。

### 例題連線錨點

- 以「矽 Z 克」矩形邊界計算兩個錨點：上邊中央與下邊中央。
- `D` 的連線終點固定為上邊中央。
- `SiC` 的連線終點固定為下邊中央。
- 所有題目、子題、步驟及顯示狀態共用同一計算，不使用題目專屬偏移。
- 先繪製連線，再繪製節點與文字，確保線段不穿過節點內容。

## 驗證

- 新增或擴充 Node 測試，先證明目前行為不符合以下條件，再實作修正：
  - 手機頁籤為 2×2 且不依賴水平溢出。
  - 倍比定律首次 HTML 內容與第一步資料一致。
  - compact renderer 的換行段落、表格與結論保有最低間距。
  - `D` 與 `SiC` 的端點分別等於「矽 Z 克」上、下邊中央。
- 執行全部既有 Node 測試與四支正式 JavaScript 語法檢查。
- 以 320、360、390、412、480 CSS px 檢查四個正式頁面及相關步驟；Canvas DPR 1、2、3 的邏輯位置必須一致。
- 更新 CSS／JavaScript 快取版本，推送 `main`，確認 GitHub Pages HTTP 200 且載入新版本。
- 提供帶快取排除參數的網址供手機實機確認；實機確認前不宣稱視覺問題已完全解決。

## 非目標

- 不重做桌機導覽設計。
- 不改寫教學內容、數值或步驟流程。
- 不重構與本次四類問題無關的 Canvas 繪圖程式。
