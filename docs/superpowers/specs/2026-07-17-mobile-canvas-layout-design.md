# 手機版 Canvas 響應式排版設計

## 目標

修正手機與社群 App 內建瀏覽器中 Canvas 文字、分子、圖表標籤互相重疊或被裁切的問題，同時維持桌機與 iPad 目前已驗證的畫面。

## 範圍

檢查並處理四個正式教學頁面及其可直接路由的所有狀態：

- `index.html`：定比定律步驟 1–8。
- `quiz.html`：例題 1 的三小題及例題 2。
- `multiple_proportions.html`：倍比定律步驟 1–10。
- `multiple_proportions_explain.html`：選擇固定元素、選擇質量、顯示結果三種狀態。

專案報告與預覽儀表板不屬於學生正式學習頁面，不修改其內容。

## 根因

Canvas 的顯示寬度會隨手機縮小，但繪圖程式仍使用桌面版固定 `rem` 字級、50–60px 固定圖表邊距、單行長文字與固定物件座標。Canvas 不會像 HTML 自動換行，因此文字超出繪圖區並與圖形重疊。

## 設計

### 共用判斷

新增無相依套件的 `canvas-responsive.js`，依 Canvas 的邏輯寬度判斷排版：

- `width <= 480`：手機緊湊排版。
- `width > 480`：完全沿用既有桌機／iPad 數值。

共用工具只提供確定性計算：緊湊模式判斷、畫布高度、圖表邊距、可容納文字的字級，以及 Canvas 文字換行。頁面仍自行決定教學內容的排列。

### 畫布尺寸

- 定比定律與倍比定律雙畫布：手機改為正方形邏輯畫布，桌機維持 4:3。
- 定比例題主圖：手機採較高的 1:0.95，桌機維持 16:13。
- 倍比例題圖表：手機採 1:0.9，桌機維持 4:3。
- CSS 繼續控制顯示寬度；JavaScript 只設定 backing store、邏輯尺寸與 DPR transform。

### 逐頁處理

- 定比定律：步驟 5–8 的標題、結論與公式改用緊湊字級／換行；第 8 步重新配置三個水分子與底部文字；圖表斜率文字分行，圖表邊距縮至可讀範圍。
- 定比例題：主圖節點、標籤與提示公式依寬度縮放；提示 Canvas 的長公式允許換行。
- 倍比定律：步驟 4–10 的雙化合物標籤、原子圖與比例說明重新分配垂直空間；圖表使用動態邊距與手機字級。
- 倍比例題：圖表邊距與座標標籤依寬度調整，數據點標籤在手機版縮短或改變對齊方向，避免超出邊界。

## 驗收條件

- 360px 與 480px Canvas 寬度下，所有長文字都能落在 Canvas 邊界內。
- 手機版文字不得與主要分子、數據點或公式重疊。
- 481px 以上仍採既有桌機／iPad 參數。
- 四頁皆使用邏輯像素與 DPR backing store，不新增 inline Canvas 顯示尺寸來源。
- 所有既有 14 項測試與新增手機排版測試通過。
- 正式部署後提供帶新版 cache key 的網址；真機截圖確認前不宣稱視覺問題完全修復。

## 非目標

- 不改教學內容、顏色、動畫順序或導覽流程。
- 不重寫 Canvas 架構、不導入框架或第三方套件。
- 不改桌機與 iPad 已通過驗證的視覺配置。
