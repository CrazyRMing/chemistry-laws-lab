# 普魯士藍介面主題設計

日期：2026-07-18

## 目標

將四個正式教學頁面的主要介面配色，由目前的橘白改為普魯士藍與白色，同時完整保留 Canvas、圖表與化學示意內部既有的資料配色。

正式頁面包括：

- `index.html`
- `quiz.html`
- `multiple_proportions.html`
- `multiple_proportions_explain.html`

## 核准的視覺方向

- 主色：普魯士藍 `#003153`
- 滑入／較亮狀態：深藍的可辨識變體，例如 `#174a6b`
- 淡色提示背景：低彩度淺藍，例如 `#eaf1f5`
- 主背景與卡片背景：白色 `#ffffff`
- 主要文字：沿用既有深灰／黑色
- 成功、錯誤與停用狀態：保留既有綠、紅、灰色

手機頁籤繼續採用已驗證的 2×2 排列；本次只改變色彩，不改變版面結構、尺寸或互動流程。

## 色彩邊界

### 納入主題更換

以下皆屬於介面層，原有橘色應改為新的主題色：

- 頁籤的作用中、滑入與焦點狀態
- 上一步、下一步、驗證答案等主要操作按鈕
- 介面框線、分隔線與主題裝飾線
- 頁面標題、區塊標題與純介面提示文字的強調色
- HTML 元素的選取、完成與提示背景
- JavaScript 動態設定的 DOM 按鈕與控制項狀態

### 明確排除

以下內容屬於教學資料或 Canvas 繪圖，不得因主題更換而改色：

- Canvas 中的資料點、曲線、座標軸、原子、分子與實驗液體
- Canvas 中用來區分資料系列或推導步驟的橘、紫、藍、綠、紅、黃色
- Canvas 中的教學標題、結論文字與提示線
- 圖例中對應資料系列的色點與線段
- 成功、錯誤、警告與停用狀態的語意色
- `assets/` 內圖片的原始配色

因此不得對 `#ff7a00`、`#f57c00`、`orange` 或 `--color-orange` 進行跨專案的盲目全域取代。

## 實作方式

### 1. 建立介面專用語意變數

在 `style.css` 的 `:root` 新增介面主題變數：

```css
--theme-primary: #003153;
--theme-primary-hover: #174a6b;
--theme-primary-soft: #eaf1f5;
--theme-on-primary: #ffffff;
```

既有 `--color-orange` 保留原值，作為仍需維持橘色的教學資料與圖表色。不得把 `--color-orange` 直接改成普魯士藍，避免連帶改變 Canvas 外的圖例與教學資料色。

### 2. CSS 介面選擇器改用主題變數

逐一將確定屬於介面的 CSS 規則改用 `--theme-*`，包括頁籤、導航、主要按鈕、區塊裝飾、焦點與滑入狀態。

每個替換都以選擇器用途判斷，不依顏色字串批次取代。與資料系列相關的圖例標記、曲線樣式及教學強調色保持不變。

### 3. JavaScript 只處理 DOM 介面狀態

部分按鈕狀態由 JavaScript 直接設定 `style.background`、`style.color` 或 `style.borderColor`。這些確定屬於 DOM 控制項的橘色改為 `var(--theme-primary)` 或對應的主題變數。

Canvas 繪圖呼叫中的 `fillStyle`、`strokeStyle`、繪圖函式顏色參數與資料系列常數不修改。實作時應依 DOM 操作與 Canvas context 操作分開檢查，禁止用同一規則處理。

### 4. 快取版本

四個正式 HTML 頁面的共用 `style.css` 版本參數同步更新。若 JavaScript 檔案內有 DOM 主題色修改，對應頁面的 script 版本參數也同步更新，確保 GitHub Pages 與手機內建瀏覽器不會繼續讀取舊快取。

## 架構與資料流

介面主題色只有一個來源：`style.css` 的 `--theme-*` 變數。HTML 元件與 JavaScript 產生的 DOM 狀態引用這些變數；Canvas 繪圖則繼續使用各繪圖檔案中的既有資料色常數或字面值。

```text
style.css --theme-* ──> HTML 介面元件
                    └─> JavaScript DOM 控制項狀態

Canvas 繪圖色常數／字面值 ──> Canvas、圖表與化學示意
```

兩條色彩路徑互不共用，避免日後再次更換網站主題時改動教學資料色。

## 相容性與錯誤處理

- CSS 自訂屬性在目前支援的 Safari、Android WebView 與主流桌面瀏覽器均可使用。
- 若主題變數未載入，CSS 規則應提供普魯士藍 fallback，例如 `var(--theme-primary, #003153)`。
- 不新增主題切換器，不增加執行期狀態或新的相依套件。
- 不調整已驗證的響應式斷點、Canvas 尺寸與手機 2×2 頁籤結構。

## 驗證方式

### 自動檢查

- 新增主題一致性測試，確認四個正式頁面載入相同的樣式版本。
- 確認 `style.css` 定義核准的四個 `--theme-*` 變數。
- 確認主要頁籤與按鈕規則引用 `--theme-primary`。
- 建立 Canvas 色彩保護測試，鎖定代表性的既有資料色與繪圖程式片段，防止主題改色誤傷。
- 執行既有全部測試與四個主要 JavaScript 語法檢查。

### 視覺檢查

- 桌面寬度：確認普魯士藍與白色主題一致，文字對比清楚。
- iPad 橫向與直向：確認框線與頁籤沒有因改色產生版面差異。
- 390px 手機寬度：確認頁籤仍為 2×2、無橫向溢位。
- 逐頁比較代表性 Canvas 與圖表，確認資料點、曲線、原子與圖例資料色和修改前相同。
- 檢查作用中、滑入、鍵盤焦點、成功、錯誤與停用狀態。

### 發布檢查

- 推送 `main` 後確認 GitHub Pages 回傳 HTTP 200。
- 確認線上 HTML 已引用新的 CSS／JavaScript 版本參數。
- 提供帶版本參數的測試網址，供 Android 內建瀏覽器與 iPad 實機確認。

## 非目標

- 不重新設計版面、字型、插圖或動畫。
- 不更換 Canvas、圖表、圖例資料系列或化學元素配色。
- 不新增深色模式或主題切換功能。
- 不重構與配色無關的 JavaScript 或 CSS。
- 不修改教學內容、步驟順序與計算邏輯。

## 完成標準

1. 四個正式頁面的介面主色一致使用普魯士藍與白色。
2. 手機頁籤維持 2×2 且沒有橫向溢位。
3. 所有 Canvas、圖表與圖例資料色保持修改前的視覺結果。
4. 成功、錯誤與停用狀態仍具有原本的語意辨識色。
5. 全部自動測試、JavaScript 語法檢查與代表性跨尺寸視覺檢查通過。
6. GitHub Pages 載入新版本資源並可供手機及 iPad 實機測試。
