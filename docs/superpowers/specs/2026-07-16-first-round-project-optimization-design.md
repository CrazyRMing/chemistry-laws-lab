# 第一輪專案優化設計

## 目標

在不改變教學流程、畫面配置、Canvas 動畫與 iPad Safari 行為的前提下，降低重複內容、過時檔案與快取設定造成的維護成本。

## 範圍

1. 移除 `PROJECT_NOTES.md` 自第二個同名標題開始的重複區段，只保留一份完整時間軸。
2. 移除未被正式頁面導覽或腳本引用的 `index_3b1b.html` 與 `app_3b1b.js`。檔案歷史繼續由 Git 保存。
3. 將 `app.js` 三張實驗圖片的 `Date.now()` 動態查詢參數改為固定版本字串，使瀏覽器可以正常快取。
4. 將四個正式頁面引用的 `style.css` 查詢版本統一為同一版本。
5. 依目前白底手繪、逐步引導、四頁式學習流程重寫 `README.md`，移除已不存在的深色毛玻璃、滑桿與碰撞模擬描述。

## 不在本輪處理

- 不抽取共用 Canvas 繪圖工具。
- 不抽取共用 Header 或版權 Modal。
- 不調整 HTML 內嵌樣式。
- 不改動教學文字、數值、互動步驟或 RWD 斷點。
- 不新增框架、套件或建置流程。

## 驗證方式

1. 新增靜態結構測試，確認：
   - 舊版副本不存在。
   - 專案筆記只有一個主標題且沒有完整區段重複。
   - 正式頁面使用相同的 `style.css` 版本。
   - `app.js` 不再以 `Date.now()` 破壞圖片快取。
   - README 描述目前的四頁學習流程，且不再宣稱使用深色毛玻璃或滑桿碰撞模擬。
2. 執行既有 iPad 版面測試。
3. 對四支正式 JavaScript 執行 `node --check`。
4. 檢查 Git diff，確保沒有修改 Canvas 版面或教學邏輯。

## 成功條件

- 所有新舊測試通過。
- 四支正式 JavaScript 語法檢查通過。
- 正式網站入口與四頁導覽路徑不變。
- 沒有新增相依套件。
